# Trade Yantra - Issue Diagnosis & Fixes
## Date: 2026-02-12

### Issues Reported
1. **Live data not streaming** - WebSocket connection not sending price updates to frontend
2. **Funds showing 0 in Live tab** - Real balance from Angel One API not displaying correctly
3. **Paper trade quantity defaulting to 1 instead of 100**

---

## Root Causes Identified

### 1. WebSocket Data Streaming Issue
**Problem**: The WebSocket connection is established, but price updates may not be reaching the frontend properly.

**Debugging Added**:
- Added debug logging in `websocket_manager.py` to track data reception every 5 seconds
- Logs will show `[WS-DEBUG]` messages when data is received from Angel One
-

 This helps verify if the backend is receiving data from the broker

**Check Required**:
- Backend must be restarted for changes to take effect
- Look for `[WS-DEBUG]` messages in backend console to confirm data flow
- If Angel WebSocket is not connecting, check:
  - JWT token validity
  - Network connectivity
  - Broker API status

---

### 2. Live Funds Showing 0

**Problem**: The Angel One `get_rms_limit()` API returns a nested data structure, but the code was expecting a flat structure.

**Fix Applied** (`backend/services/live_service.py`):
- Enhanced `get_funds()` method with robust parsing
- Now handles multiple response patterns from Angel RMS API:
  - Pattern 1: `{'data': {'net': 123456}}`
  - Pattern 2: `{'net': 123456}`
  - Pattern 3: `{'availablecash': 123456}`
- Added comprehensive debug logging to see raw API response
- Returns `{'net': balance, 'raw': full_response}` to frontend

**Expected Result**:
- Frontend will now receive proper balance value
- Debug logs will show:
  ```
  [DEBUG] RMS Data Response: {full API response}
  [DEBUG] Extracted Net Balance: 123456.78
  ```

---

### 3. Paper Trade Quantity Defaulting to 1 Instead of 100

**Problem**: The `trade_quantity` attribute was not initialized in the `Session` class and not persisted correctly.

**Fixes Applied**:

1. **Session Initialization** (`backend/services/session_manager.py`):
   - Added `self.trade_quantity = 100` in `Session.__init__`
   - Ensures new sessions default to 100 quantity

2. **Persistence** (`backend/services/persistence_service.py`):
   - Added `"trade_quantity": getattr(session, 'trade_quantity', 100)` in `save_session()`
   - Ensures quantity is saved and loaded from JSON/GCS

3. **Session Restoration** (`backend/services/session_manager.py`):
   - Added `session.trade_quantity = existing_data.get('trade_quantity', 100)` in `create_session()`
   - Added same in `get_session()` recovery logic

4. **Trade Execution** (`backend/services/websocket_manager.py`):
   - Changed default fallback from `1` to `100` in `get_trade_quantity()` function
   - Also fixed default in `buffer_pct` (was 0.25, now correctly 0.45%)

5. **Paper Service** (`backend/services/paper_service.py`):
   - Added safety checks: `if qty == 0: qty = 100` in averaging and closing logic
   - Prevents edge cases where quantity might be 0
